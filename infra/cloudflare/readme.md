# Cloudflare Tunnel Infrastructure

This component manages the Cloudflare Tunnel (formerly Argo Tunnel) for the homelab. Its primary purpose is to securely expose the internal Kubernetes Ingress controller to the public internet without opening inbound ports on the firewall.

## Architecture
The `cloudflared` deployment runs in the cluster, connects outbound to Cloudflare, and routes traffic for `hungops.tech` and `*.hungops.tech` to the internal `ingress-nginx-controller`.

## Prerequisites & Manual Setup

This component requires several manual steps to establish the tunnel and provide credentials before the GitOps deployment can function correctly.

### 1. Authenticate Cloudflared
Login to your Cloudflare account from your local machine:
```bash
cloudflared tunnel login
```

### 2. Create the Tunnel
Create a new tunnel specifically for the homelab:
```bash
cloudflared tunnel create hungops-homelab
```
Note the **Tunnel ID** generated by this command.

### 3. Locate Credentials
The credentials JSON file will be saved on your local machine at:
```bash
~/.cloudflared/<TUNNEL_ID>.json
```

### 4. Prepare Kubernetes Namespace
Ensure the `cloudflare` namespace exists:
```bash
kubectl create namespace cloudflare
```

### 5. Create Kubernetes Secret
Create the secret containing the tunnel credentials. This secret is referenced by the deployment in `manifest.yaml`.

Replace `<TUNNEL_ID>` with your actual Tunnel ID:
```bash
kubectl create secret generic tunnel-credentials \
  --from-file=credentials.json=$HOME/.cloudflared/<TUNNEL_ID>.json \
  -n cloudflare
```

### 6. Configure DNS Records
Finally, add CNAME records in the Cloudflare Dashboard to point your domain and subdomains to the tunnel.

1. Go to your **Cloudflare Dashboard**.
2. Navigate to **DNS > Records**.
3. Add the following records:
   - **Type**: `CNAME`
   - **Name**: `@` (for `hungops.tech`)
   - **Target**: `<TUNNEL_ID>.cfargotunnel.com`
   - **Proxy status**: Proxied (Orange cloud)

   - **Type**: `CNAME`
   - **Name**: `*` (for `*.hungops.tech`)
   - **Target**: `<TUNNEL_ID>.cfargotunnel.com`
   - **Proxy status**: Proxied (Orange cloud)

## Configuration Note
The `tunnel` ID in `manifest.yaml` (within the `ConfigMap`) must match the ID of the tunnel you created in step 2.

```yaml
# manifest.yaml
data:
  config.yaml: |
    tunnel: <YOUR_TUNNEL_ID_HERE>
    credentials-file: /etc/cloudflared/creds/credentials.json
```
